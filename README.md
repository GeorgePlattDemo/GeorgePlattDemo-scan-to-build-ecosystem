Public review repository for the Scan-to-Build / Home as Twin architecture: an owner-held home project record, governed WorkPackets, intelligent routing, bounded local fabrication, workforce-aligned operation, and outcome preservation.

Connected to U.S. Patents 9,720,401 and 10,768,609: method and system for consumer home projects ordering and fabrication.

Core Thesis

Scan-to-Build reduces unnecessary atom travel by allowing precise, governed data to travel first.

The homeowner holds a persistent project record. Project information is captured, standardized, validated, packetized, routed, fulfilled, and preserved. Each completed outcome returns to the homeowner-controlled record, so future projects start from better information than the last.

Core Invariants
The homeowner controls the record.
Capture is standardized.
Routing is intelligent, explainable, and governed.
Outcomes are preserved and returned to the homeowner record.

What the Package Shows

The package is organized around one claim: a homeowner-controlled record can coordinate home projects more reliably when project information is captured once, resolved into governed packets, routed through eligible channels, and returned to the record as preserved outcomes.

The public demo explains the Scan-to-Build ecosystem.

The Home as Twin demo shows the homeowner-held project record as a coordination surface.

The architecture document explains the production reference model.

The ontology document explains the shared language for materials, labor tiers, machine envelopes, refusals, and outcomes.

The WorkPacket examples show that the packet is a concrete, machine-readable contract, not only a concept.

The workforce brief explains the operator side of the system: bounded fabrication cells, operator judgment, material handling, safety boundaries, and workforce alignment.

Proof Cases
Woodwork: fabrication proof

Woodwork is the fabrication proof:

capture → packet → material validation → machine envelope → local fabrication → labeled output → outcome

This proof case shows the path toward bounded local fabrication.

Windows: orchestration proof

Windows is the orchestration proof:

research → configured scope → vendor packet → quote comparison → consented purchase → fulfillment → outcome

This proof case shows how the same homeowner-held record can coordinate a vendor-manufactured project.

The two proof cases are different on purpose. Woodwork shows the path toward local fabrication. Windows shows how the same record can govern a project that routes through outside vendors and fulfillment channels.

In both cases, the homeowner-controlled record produces a versioned packet, the packet routes through eligible channels, refusals are preserved, and the outcome returns to the record.
